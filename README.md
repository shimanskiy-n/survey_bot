# Простой Telegram-бот на Python (aiogram + SQLite)

Этот проект — минимальный пример Telegram-бота на `aiogram`, который:

- **При старте (`/start`)**: если пользователь ещё не зарегистрирован, бот попросит **ФИО** и сохранит его в SQLite (привязав к Telegram ID).
- После регистрации (и при повторном `/start`) бот выводит приветствие с **ФИО** и задаёт пользователю **все активные вопросы** по порядку.
- **Ответы** сохраняются в отдельную таблицу `answers` (указывается Telegram ID пользователя и ID вопроса).

### Стек

- **Язык**: Python 3.10+
- **Библиотека для бота**: `aiogram 3`
- **База данных**: SQLite

---

### Структура проекта

- **`app/`** — исходный код бота  
  - **`__init__.py`** — делает папку модулем Python  
  - **`config.py`** — загрузка настроек (токен бота)  
  - **`db.py`** — работа с базой данных SQLite  
  - **`handlers.py`** — обработчики команд и сообщений  
  - **`main.py`** — точка входа, запуск бота  
- **`requirements.txt`** — зависимости проекта  
- **`README.md`** — документация проекта

### Установка

1. **Клонировать / скопировать проект** (или просто создать папку и файлы как здесь).

2. **Создать и активировать виртуальное окружение (рекомендуется)**:

```bash
python -m venv .venv
.venv\Scripts\activate
```

3. **Установить зависимости**:

```bash
pip install -r requirements.txt
```

4. **Создать Telegram-бота и получить токен** у [BotFather](`https://t.me/BotFather`).

5. **Указать токен бота** через переменную окружения:

```bash
set BOT_TOKEN=ВАШ_ТОКЕН_ОТ_BOTFATHER
```

На Windows PowerShell:

```powershell
$env:BOT_TOKEN="ВАШ_ТОКЕН_ОТ_BOTFATHER"
```

> В коде токен читается из переменной окружения `BOT_TOKEN`. Файл `.env` вы можете использовать сами, но он уже добавлен в `.gitignore`, чтобы не попасть в репозиторий.

### Структура данных в базе (SQLite)

Используется один файл базы данных `bot.db` в корне проекта.

Таблица `users`:

- `id` — целочисленный первичный ключ
- `telegram_id` — Telegram ID пользователя (уникальный)
- `full_name` — строка с ФИО
- `is_admin` — признак администратора (0/1)

Таблица `questions`:

- `id` — целочисленный первичный ключ
- `question_text` — текст вопроса
- `is_active` — признак активности (0/1)
- `sort_order` — порядок показа активных вопросов

Таблица `answers`:

- `id` — целочисленный первичный ключ
- `telegram_id` — Telegram ID пользователя
- `question_id` — ID вопроса
- `answer_text` — текст ответа пользователя

Таблицы создаются автоматически при старте бота.

---

### Запуск бота

Из корня проекта:

```bash
python -m app.main
```

После запуска:

1. Откройте вашего бота в Telegram.
2. Напишите ему команду `/start`.
3. Если вы в базе ещё не зарегистрированы — бот попросит ввести **ФИО** (одной строкой).
4. Бот задаст **все активные вопросы** по порядку.
5. Вы отправите ответы, и они сохранятся в базе.

### Где доработать под себя

- **Тексты вопросов** — через админ-команды (см. ниже), таблица `questions`.
- **Добавить новые поля в БД** — файл `db.py` (структура таблиц) и `handlers.py` (логика сохранения).
- **Изменить язык сообщений** — всё текстовое общение с пользователем находится в `handlers.py`.

### Админка (создание/деактивация вопросов)

Администратор — пользователь из таблицы `users` с `is_admin = 1`.

Как стать админом:
- если задано окружение `ADMIN_TELEGRAM_ID`, то пользователь с этим Telegram ID будет админом
- иначе первый зарегистрированный пользователь станет админом

Админ-команды:

- `/list_questions` — показать список вопросов с `id` и статусом
- `/add_question <текст>` — добавить новый вопрос (активный)
- `/deactivate_question <id>` — пометить вопрос неактивным
